# PRD: EMOTODO(이모토도)

> 미루기 습관과 학습된 무기력을 극복하기 위한 자기 인식 기반 기록 앱

---

## 1. 문제 정의

### 1.1 핵심 문제
사용자는 중요한 일(특히 공부)을 지속적으로 미루고 있으며, 이로 인해 학습된 무기력과 정체성 문제를 겪고 있다.

### 1.2 문제의 구조

```
[감정 저하] → [도피 행동 (넷플릭스, 게임)] → [인식은 하지만...]
     ↓
["시간이 부족해" 생각] → [포기] → [다시 도피] → [죄책감 누적]
```

### 1.3 사용자가 발견한 통찰
- **감정이 행동을 이끈다**: 무기력하면 소비적 행동(도파민 즉시 획득)으로 도피
- **인식하면 달라진다**: 의식적으로 상황을 알 때는 행동 가능
- **"제대로 해야 한다"는 생각이 장벽**: 평일 30분을 못하면 → 주말 4시간 부담 → 더 미루게 됨

### 1.4 미루는 것 vs 미루지 않는 것

| 미루지 않음 | 이유 |
|-------------|------|
| 집안일 | 끝이 보임 + 동거인(외부 압박) |
| 업무 | 회사라는 환경이 강제 |
| 운동 | 몸의 신호가 명확 |

| 미룸 | 이유 |
|------|------|
| 공부 | 끝이 안 보임 + 외부 강제 없음 + "제대로" 기준 |

**공부의 특성**: 중요하지만 긴급하지 않고, 끝도 없고, 나만 알고 있는 일

## 2. 제품 목표

### 2.1 핵심 가치
> **"실패/성공 판정이 아니라, 오늘의 나를 이해하는 기록"**

### 2.2 구체적 목표
1. 작은 단위(30분)를 지키면 누적 부담(주말 4시간)이 줄어드는 걸 체감하게
2. 못한 날이 쌓여도 죄책감이 아니라 **패턴 인식**으로 이어지게
3. 꾸준히 기록하고 해나갈 수 있는 습관 형성
4. 스트레스 없이 해야 할 일을 해낼 수 있게

### 2.3 안티 목표 (하지 않을 것)
- ❌ 실패했을 때 벌칙이나 부정적 피드백
- ❌ 복잡한 할 일 관리 (기존 투두앱과 경쟁 X)
- ❌ 게이미피케이션으로 또 다른 도파민 중독 유발

## 3. 타겟 사용자

### 3.1 페르소나
- 해야 할 일을 알고 있지만 시작하지 못하는 사람
- 자기 비판이 심해서 "실패"를 기록하기 두려운 사람
- 기존 할 일 앱에서 미완료 항목에 지쳐본 경험이 있는 사람
- 자기 이해와 패턴 인식에 관심 있는 사람

### 3.2 핵심 사용 맥락
- **공부** (영어, CS 지식, 개인 프로젝트 등)
- 끝이 명확하지 않은 장기적 자기계발

## 4. 핵심 기능 (MVP)

### 4.1 화면 구성

| 화면 | 역할 |
|------|------|
| **홈** | 오늘 할 일 목록 + 퀵 타이머 버튼 (5분/10분/15분) |
| **할 일 추가** | 할 일 + 예상 시간 입력 (최소 단위 권장) |
| **타이머** | 카운트다운 + 일시정지/완료 |
| **기록하기** | 했음/못했음 + 감정·이유 기록 |
| **히스토리** | 과거 기록 열람 |
| **주간 리뷰** | 패턴 시각화 (Phase 2) |

### 4.2 기능 상세

#### 4.2.1 홈 화면 (오늘의 할 일)

**할 일 카드 구성:**
```
┌─────────────────────────────────────┐
│  📚 영어 공부                        │
│  예상 30분                           │
│                                     │
│  [5분만?] [10분만?] [15분만?]        │  ← 퀵 타이머 넛지
│                                     │
│  [완료] [부분완료] [미룸]            │  ← 상태 기록
└─────────────────────────────────────┘
```

**UX 의도:**
- 할 일을 보자마자 "작게 시작하기" 옵션이 눈에 들어옴
- "30분 다 해야지"가 아니라 "5분만 해볼까?" 마인드셋 유도
- 탭하면 바로 해당 시간으로 타이머 시작

**상태 표시 (3단계):**
- 아직 안 함: 퀵 타이머 버튼 + 상태 버튼 표시
- 완료: 연한 파랑 배경 + "완료" 뱃지
- 부분완료: 연한 보라 배경 + "부분완료" 뱃지
- 미룸: 연한 회색 배경 + "미룸" 뱃지

#### 4.2.2 할 일 추가
#### 4.2.2 할 일 추가
```
- 할 일 이름 (예: "영어 공부")
- 예상 소요 시간 (기본값: 30분, 최소 권장)
- 반복 설정 (매일 / 특정 요일)
```

**UX 원칙**: 
- "30분이면 충분해요" 같은 격려 메시지
- 큰 시간 입력 시 "쪼개볼까요?" 제안

#### 4.2.3 상태 기록 (완료 / 부분완료 / 미룸)

**완료:**
```
✓ 완료
→ "어떤 기분이에요?" (감정 선택)
   - 뿌듯해요 / 후련해요 / 힘들었어요 / 자랑스러워요 / 그냥 그래요
```

**부분완료:**
```
△ 부분완료
→ "어떤 기분이에요?" (감정 선택)
   - 그래도 했어요 / 조금이라도 해서 다행 / 더 하고 싶었어요 / 힘들었어요 / 그냥 그래요
```

**미룸:**
```
− 미룸
→ "어떤 기분이에요?" (감정 선택)
   - 피곤했어요 / 마음이 무거워요 / 괜찮아요 / 바빴어요 / 특별한 감정 없어요
→ "왜 그랬을까요?" (이유 선택, 비판 없는 톤)
   - 너무 피곤했어요 / 시간이 부족했어요 / 깜빡했어요 / 기분이 안 났어요 / 어렵게 느껴졌어요 / 다른 일이 생겼어요
```

**핵심 UX**:
- "실패"라는 단어 사용 금지
- 기록 자체를 칭찬 ("기록했네요!")
- 부분완료를 긍정적으로 인정 ("조금이라도 한 게 대단해요!")
- 미룸 이유도 **데이터**로 취급 (패턴 인식용)

#### 4.2.4 타이머

> **핵심 의도**: "시간이 부족하다"는 심리적 장벽을 낮춰서 일단 시작하게 만들기

**진입점:**
- 할 일 항목에 "시작하기" 버튼
- 예상 시간이 설정된 경우에만 활성화

**타이머 옵션:**
```
"얼마나 할까요?"

[5분] [15분] [30분] [전체 시간]
       ↑
   기본 강조 (낮은 진입장벽)
```

**타이머 진행 중:**
- 심플한 카운트다운 표시
- 일시정지/재개 가능
- 중간에 그만둬도 OK ("N분 했어요" 기록)

**타이머 종료 시:**
```
"5분 완료! 🎉"

[더 할래요] → 추가 시간 선택
[여기까지] → 기록 화면으로 이동
```

**UX 원칙:**
- "5분만"을 가장 눈에 띄게 — 진입장벽 최소화
- 중간에 멈춰도 **한 것으로 인정** (부분 완료)
- 예상 시간 다 못 채워도 괜찮다는 메시지

**데이터 기록:**
- 실제 집중 시간 (분)
- 타이머 사용 여부
- 중간 포기 vs 끝까지 완료

#### 4.2.5 히스토리
- 날짜별 기록 열람
- 상태(완료/부분완료/미룸) + 감정/이유 함께 표시

#### 4.2.6 주간 리뷰 (Phase 2)
- 요일별 완료율 시각화
- 자주 나오는 "못한 이유" 패턴 표시
- "에너지가 없었던 날" vs "완료한 날" 비교

```typescript
// 할 일
interface Task {
  id: string;
```

## 7. 핵심 UX 원칙

### 7.1 언어 톤
| 피해야 할 표현 | 사용할 표현 |
|----------------|-------------|
| 실패 | 오늘은 못했네요 |
| 아직 안 함 | 아직 기록 전 |
| 왜 안 했어요? | 왜 그랬을까요? |
| 열심히 하세요 | 작게 시작해봐요 |

### 7.2 피드백 철학
- **했을 때**: 담담하게 인정 ("기록 완료")
- **못했을 때**: 이유를 묻되 판단하지 않음
- **기록 자체**: 가장 큰 성공으로 취급

### 7.3 시각적 표현
- 완료/미완료를 빨강/초록 같은 강한 대비로 표현하지 않음
- 부드러운 색상 (예: 완료=연한 파랑, 미완료=회색)
- 미완료가 "나쁜 것"처럼 보이지 않게

## 9. 성공 지표

### 9.1 정성적
- 앱 사용 후 "실패감" 감소
- 작은 단위로 꾸준히 기록하는 습관 형성
- 자기 패턴에 대한 인식 증가

### 9.2 정량적 (추후)
- 주간 기록 일수
- 30분 단위 완료 비율 변화
- "시간 부족" 이유 감소 추이

## 10. 열린 질문

1. **앱 이름**: "오늘의 나"? 다른 후보?
2. **온보딩**: 첫 실행 시 사용자의 상황을 물어볼까?
3. **알림 전략**: 알림이 오히려 부담이 될 수 있음 → 옵트인?
4. **외부 공유**: 친구와 함께 쓰는 기능 필요?
